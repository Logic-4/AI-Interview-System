AI Mock Interview Training System Workflow1
1
Candidate on-boarding and structured input captureWhat happens: The candidate creates an account (or uses guest mode) and fills a structured “Interview Setup” form.
Candidate inputs (core):
Target job position (e.g., “Junior Backend Developer”)
Job description (paste text or upload PDF/text)
Experience level (Intern/Junior/Mid/Senior)
Interview type (HR / Technical / Behavioral / Mixed)
Preferred skills to focus (pick from list + free text)
Interview length (10/20/30/45 mins)
Language preference (optional)
Seniority goal (optional: “aim for mid-level questions”)
System output:
Creates an Interview Context object (this is the “brain” of the session)
Stores it under the candidate profile and the upcoming session

2
Job description parsing and profile building (AI preparation step)What happens: The system turns the job description into a structured role profile.
AI extracts:
Key skills (tools, frameworks, soft skills)
Core responsibilities
Common scenarios (role tasks)
“Must have” vs “nice to have”
Keywords and phrases to look for in answers
System output:
Role Profile (derived from JD)
Rubric signals (what good answers should include)
___________________________________________________3
3
Question generation and interview plan creationWhat happens: The system generates a question set and a plan before the interview starts.
Inputs used:
Interview Context (position, level, type, length)
Role Profile (skills + responsibilities from JD)
Question bank/templates (if available)
Randomization + difficulty rules
Candidate history (avoid repeats, target weak areas)
Process:
Decide interview structure based on type:
HR: intro, motivation, strengths/weakness, culture fit
Technical: core skills, scenario tasks, debugging, fundamentals
Behavioral: STAR-based situation questions
Mixed: balanced sections
Choose number of questions by time (example: 20 mins → ~6–8 questions)
Assign each question a “goal”:
Test React hooks
Test teamwork conflict
Test REST API knowledge
      4.  Generate questions (template-based + AI-generated)
      5. Generate follow-up question rules (probing if answer is vague)
Outputs:
Interview Plan (ordered questions, section breakdown, difficulty per question)
Scoring rubric per question (what the answer should contain)
Live interview session (voice simulation + interaction loop) What happens: The AI runs the interview in real time using the plan.4
4
Loop per question:
AI asks the question (voice + text on screen).
Candidate answers verbally (recording starts).
Candidate ends answer (silence detect or “Done”).
System saves audio for this question.
AI may ask a follow-up based on rules:
If answer lacks details → “Can you give an example?”
If answer is wrong → “Explain your approach step-by-step.”
Continue to next question until time ends.
Outputs during session:
Audio recordings per answer
Real-time transcript (optional live)
Instant feedback highlights (optional, after each answer)5
5
Scoring pipeline (evaluation happens per answer, then overall)What happens: Each answer is processed through modules, then    combined into scores.
Step-by-step per answer:
Speech-to-Text:
Convert audio → transcript + timestamps + confidence
Content scoring (job-aware):
Relevance: did it answer the question and match JD needs?
Coverage: did it mention key skills/responsibilities?
Correctness (for technical): is reasoning valid?
Communication scoring:
Clarity: structured answer, less rambling, direct response
Fluency: pace, pauses, smoothness
Confidence signals: hesitation, uncertainty phrases, unstable volume
Grammar/sentence structure (from transcript)
Optional coaching:
Pronunciation flags (words misheard often)
Stress indicators (presented as “signals”)
Then combine:
Per-question scores + notes
Section scores (HR vs technical vs behavioral)
Overall session score (weighted by interview type)
Outputs:
Scored answers
Highlighted transcript (good parts + weak parts)
Audio markers (pause points, filler words)
___________________________________________________6
6
Feedback generation (actionable, not generic)What happens: The system turns raw scores into clear feedback the candidate can use.
Per question feedback:
What went well (1–2 points)
What to fix (2–3 points)
Suggested improved answer structure (short)
Missing keywords/ideas from JD (if relevant)
A short “better version” sample (optional, not too long)
Session-level feedback:
Top strengths (based on consistent high scores)
Top weaknesses (repeat patterns)
“Do this next” plan (3–5 actions)
Suggested practice questions based on weak areas
Outputs:
Full session report (PDF/web view)
Downloadable summary
Recommendations list
__________________________________________________7
7
Practice and repetition loop (improve using your own data)What happens: Candidates can immediately practice weak areas.
Options:
Retry the same question now
Practice a set of “weak area” questions
Run the same template again with randomization
The system uses:
The exact feedback from the last session
Targets like “reduce fillers”, “mention JD skill X”, “use STAR”
Outputs:
Before vs After comparisons
Improvement score changes
___________________________________________________8
8
Progress dashboard and long-term trackingWhat happens: Every session updates the candidate’s history and shows growth.
Dashboard views:
Overall score trend (weekly/monthly)
Subscores trend (clarity, relevance, fluency, confidence, grammar)
Role-based history (performance for “Frontend” vs “Support”)
Common mistakes list (top repeated issues)
Keyword coverage trend (how well answers match JD needs)
Admin view (optional):
System usage stats
Average scores by role template
Common weak areas across users (for improving templates)
Outputs:
Progress charts and stats
Personalized next-session plan
“Ready level” estimate (optional: “close to Junior interview standard”)
